# Contributing

Thanks for your interest in contributing to Smart Tab Organizer! Small, focused contributions are welcome.

## Reporting issues

- Open an issue describing the bug or feature request.
- Include steps to reproduce, your browser (Chrome/Edge) and version, and any relevant console errors.

## Development workflow

- Fork the repository and create a branch: `feature/my-feature` or `fix/issue-123`.
- Make changes and test locally by loading the extension as an unpacked extension (see README).
- Create a pull request with a concise description of the change.

## Code style

- Keep code style consistent with existing files (vanilla JS, minimal dependencies).
- Avoid large unrelated refactors in the same PR.

## Testing locally

1. Open `chrome://extensions/` (or `edge://extensions/`).
2. Enable *Developer mode*.
3. Click **Load unpacked** and select this repository folder.
4. Use the options page and popup to validate behavior.

## Release notes

- Add an entry to `RELEASE_NOTES.md` (or the PR description) summarizing user-visible changes.

## Contact

If you have questions or need help testing, contact the maintainer listed in `README.md`.
